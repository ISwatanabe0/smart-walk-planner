# SmartWalk 画面設計書

## 1. 概要
本書は、SmartWalk における画面構成、画面項目、画面遷移、画面ごとの入出力を定義する。

SmartWalk は、利用者の現在地または指定地点を起点として、好みに応じた散歩ルートを生成・表示する Web アプリケーションである。

---

## 2. 画面一覧

| 画面ID | 画面名 | 概要 |
|---|---|---|
| SCR-001 | トップ / ルート検索画面 | 出発地点や条件を入力してルート生成を行う |
| SCR-002 | ルート結果画面 | 生成されたルートを地図上に表示し、詳細を確認する |
| SCR-003 | エラー / 再試行案内表示 | API失敗や位置情報取得失敗時の案内表示 |
| SCR-004 | 読み込み表示 | ルート生成中のローディング表示 |

---

## 3. 画面遷移

```text
[SCR-001 トップ / ルート検索画面]
   ├─ ルート生成成功 → [SCR-002 ルート結果画面]
   ├─ ルート生成失敗 → [SCR-003 エラー表示]
   └─ ルート生成中   → [SCR-004 読み込み表示]

[SCR-002 ルート結果画面]
   ├─ 条件変更 → [SCR-001 トップ / ルート検索画面]
   ├─ 再検索   → [SCR-004 読み込み表示]
   └─ API失敗 → [SCR-003 エラー表示]

[SCR-003 エラー表示]
   ├─ 再試行 → [SCR-004 読み込み表示]
   └─ 戻る   → [SCR-001 トップ / ルート検索画面]

---

## 4. 画面詳細

### 4.1 SCR-001 トップ / ルート検索画面

#### 4.1.1 目的
利用者が散歩ルート生成条件を入力し、ルート検索を実行する。

#### 4.1.2 主な機能
- 現在地取得 / 手動入力
- 距離・時間・好み条件の入力
- ルート生成実行およびバリデーション

#### 4.1.3 画面項目一覧

| 項番 | 項目名 | UI種別 | 必須 | 入力/表示 | 内容 |
|---|---|---|---|---|---|
| 1 | アプリタイトル | ラベル | - | 表示 | SmartWalk タイトル表示 |
| 2 | 地図エリア | 地図 | - | 表示 | OpenStreetMap 表示 |
| 3 | 現在地取得ボタン | ボタン | - | 入力 | ブラウザ位置情報取得 |
| 4 | 出発地点入力欄 | テキストボックス | 条件付き必須 | 入力 | 現在地取得不可時の手動指定 |
| 5 | 目標距離入力 | 数値入力 | 任意 | 入力 | km または m 単位 |
| 6 | 目標時間入力 | 数値入力 | 任意 | 入力 | 分単位 |
| 7 | 景観重視 | チェックボックス | 任意 | 入力 | 景観の良いルートを優先 |
| 8 | 信号回避 | チェックボックス | 任意 | 入力 | 信号が少ないルートを優先 |
| 9 | 大通り回避 | チェックボックス | 任意 | 入力 | 幹線道路を避ける |
|10 | 観光名所経由 | チェックボックス | 任意 | 入力 | 観光スポット付近を通る |
|11 | 周回ルート | チェックボックス/トグル | 任意 | 入力 | 出発地点へ戻るルートを優先 |
|12 | ルート生成ボタン | ボタン | - | 入力 | 条件に基づいて探索開始 |
|13 | 入力エラーメッセージ | メッセージ | - | 表示 | 入力不備時に表示 |

#### 4.1.4 入力チェック

| 項目 | チェック内容 | エラーメッセージ例 |
|---|---|---|
| 出発地点 | 現在地未取得かつ手動入力未指定ならエラー | 出発地点を指定してください |
| 目標距離 / 目標時間 | 両方未入力の場合はデフォルト値適用またはどちらか必須 | 距離または時間を指定してください |
| 目標距離 | 0以下は不可 | 距離は正の値を入力してください |
| 目標時間 | 0以下は不可 | 時間は正の値を入力してください |

#### 4.1.5 ボタン押下時処理
- **現在地取得ボタン**
  - ブラウザ Geolocation API 呼び出し
  - 成功: 現在地を内部保持し地図中心を移動
  - 失敗: エラーメッセージ表示
- **ルート生成ボタン**
  - 入力値バリデーション
  - 条件を API リクエスト形式へ変換
  - ルート生成 API 呼び出し
  - 成功: SCR-002 へ遷移
  - 失敗: SCR-003 を表示

#### 4.1.6 画面イメージ
```
+---------------------------------------------------+
| SmartWalk                                         |
+---------------------------------------------------+
| [地図表示エリア]                                  |
|                                                   |
|                                                   |
+---------------------------------------------------+
| [現在地取得]  出発地点: [_____________________]   |
| 距離: [____] km   時間: [____] 分                 |
|                                                   |
| [ ] 景観重視                                      |
| [ ] 信号回避                                      |
| [ ] 大通り回避                                    |
| [ ] 観光名所経由                                  |
| [ ] 周回ルート                                    |
|                                                   |

+---------------------------------------------------+
```

### 4.2 SCR-002 ルート結果画面

#### 4.2.1 目的
生成された散歩ルートを確認し、Google Maps 連携や再検索を行う。

#### 4.2.2 主な機能
- ルートポリライン表示
- 距離・推定時間・特徴の表示
- 経由スポット一覧表示
- Google Maps 遷移 / 条件変更 / 再検索

#### 4.2.3 画面項目一覧

| 項番 | 項目名 | UI種別 | 入力/表示 | 内容 |
|---|---|---|---|---|
| 1 | 地図エリア | 地図 | 表示 | ルートをポリライン表示 |
| 2 | 出発地点マーカー | 地図マーカー | 表示 | 開始地点を表示 |
| 3 | 終了地点マーカー | 地図マーカー | 表示 | 終了/周回帰着点を表示 |
| 4 | 経由地点マーカー | 地図マーカー | 表示 | 観光地やPOIを表示 |
| 5 | 総距離 | ラベル | 表示 | 例: 4.2km |
| 6 | 推定時間 | ラベル | 表示 | 例: 58分 |
| 7 | ルート特徴 | ラベル | 表示 | 景観重視、信号少なめ等 |
| 8 | 経由スポット一覧 | リスト | 表示 | 公園や観光地名称 |
| 9 | Google Mapsで開く | ボタン | 入力 | Google Maps へ遷移 |
|10 | 条件変更 | ボタン | 入力 | 検索条件編集へ戻る |
|11 | 再検索 | ボタン | 入力 | 同条件で再生成/別候補取得 |

#### 4.2.4 表示項目詳細
- **ルート特徴**: 景観スコア、信号回避率、幹線道路回避率、経由スポット数を表示。
- **経由スポット一覧**: 表示例「大通公園」「中島公園」「時計台周辺」「豊平川沿い歩道」。

#### 4.2.5 ボタン押下時処理
- **Google Mapsで開く**: ルート情報から Google Maps URL を生成し、新規タブまたはアプリ連携で起動。
- **条件変更**: 検索条件を保持したまま SCR-001 へ戻る。
- **再検索**: 同条件で再度ルート探索し、候補ルートを再取得。成功時は画面更新。

#### 4.2.6 画面イメージ
```
+---------------------------------------------------+
| ルート結果                                        |
+---------------------------------------------------+
| [地図表示エリア（ルート描画）]                    |
|                                                   |
+---------------------------------------------------+
| 総距離: 4.2km   推定時間: 58分                    |
| 特徴: 景観重視 / 信号少なめ / 大通り回避          |
|                                                   |
| 経由スポット                                      |
| - 大通公園                                        |
| - 豊平川河川敷                                    |
|                                                   |
| [Google Mapsで開く] [条件変更] [再検索]           |
+---------------------------------------------------+
```

### 4.3 SCR-003 エラー / 再試行案内表示

#### 4.3.1 目的
エラー内容を利用者へ通知し、再試行または条件見直しを促す。

#### 4.3.2 想定エラー
- 位置情報取得拒否
- Overpass API 取得失敗
- 条件に合うルートなし
- サーバ内部エラー

#### 4.3.3 表示項目一覧

| 項番 | 項目名 | UI種別 | 内容 |
|---|---|---|---|
| 1 | エラータイトル | ラベル | エラー概要 |
| 2 | エラー詳細メッセージ | メッセージ | 利用者向け説明 |
| 3 | 再試行ボタン | ボタン | 再度処理実行 |
| 4 | 条件見直しボタン | ボタン | SCR-001 へ戻る |

#### 4.3.4 エラーメッセージ例

| エラー種別 | メッセージ |
|---|---|
| 現在地取得失敗 | 位置情報を取得できませんでした。手動で出発地点を指定してください。 |
| API失敗 | ルート情報の取得に失敗しました。時間をおいて再試行してください。 |
| 条件不一致 | 条件に合うルートが見つかりませんでした。条件を緩めて再検索してください。 |

### 4.4 SCR-004 読み込み表示

#### 4.4.1 目的
ルート探索中であることを明示し、利用者の待機不安を軽減する。

#### 4.4.2 表示内容
- スピナー
- 「ルートを作成中です…」メッセージ
- 長時間時の補助文言「道路情報や景観ポイントを確認しています」

---

## 5. レスポンシブ対応方針
- **PC**: 地図を広く表示し、条件入力欄はサイドバーまたは上部に配置。
- **スマートフォン**: 地図を上部に、条件入力は折りたたみ式パネル。Google Maps 連携ボタンは下部固定も検討。

## 6. UI/UX 方針
- 初回利用者でも迷わないシンプルな操作。
- 位置情報失敗時の代替導線を明確化。
- 結果画面で「なぜこのルートなのか」を説明。
- ボタン文言は行動ベースで統一（例: 「検索」ではなく「ルート生成」）。
